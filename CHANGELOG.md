# 更新日志

本项目遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 结构，并使用语义化版本号。

## [Unreleased]

### 新增

- 账号自定义字段支持填写、编辑、敏感值遮罩和删除确认。
- 2FA 支持编辑，并完整保留 URI 中的算法、位数、周期和 HOTP 计数器。
- 回收站支持恢复或永久删除服务信息。
- 侧栏统一使用“回收站”命名，并明确同时包含账号和服务。
- 服务信息支持 URL、备注、关联主账号、详情收藏和真实拖拽排序。
- 数据库导入前增加 JSON 结构与 SQLite 完整性验证。
- 导入、导出和 CSV 操作增加结果反馈与错误提示。
- CSV 导入支持 OTP URI 参数、关联 2FA 卡片、手机号和恢复邮箱，并报告无效或不可识别行。
- 增加离线 CSP、渲染沙箱和外部导航限制，移除启动时的第三方字体请求。

### 修复

- 修复加密用户名无法被账号搜索命中的问题。
- 修复敏感账号自定义字段明文展示及更新时可能降级为明文的问题。
- 修复账号密码、2FA 密钥和服务敏感字段进入编辑状态后默认明文展示的问题。
- 修复 TOTP 密钥在数据库和 JSON 备份中以明文保存的问题。
- 修复 JSON 恢复丢失 2FA 与账号关联的问题。
- 修复导入 SQLite 后服务信息仍使用旧数据库连接的问题。
- 修复 WAL 模式下直接复制数据库可能遗漏最新记录的问题。
- 修复取消账号编辑后未保存内容再次出现的问题。
- 修复编辑账号时切换账号或新建账号会静默丢失未保存内容的问题。
- 修复编辑自定义字段时切换账号或页面会静默丢失未保存内容的问题。
- 未实际修改内容时不再弹出“放弃未保存修改”确认，并禁用无变化的保存操作。
- 关闭应用窗口时若仍有未保存账号或自定义字段修改，会先请求确认。
- 修复服务使用名称、更新时间等自动排序时仍可拖动但顺序看似不生效的问题。
- 服务、字段和分组删除统一使用应用内确认框，网址打开失败改用非阻塞消息提示。
- 修复“其他”类型账号被默认命名为 Microsoft 账号的问题。
- 修复账号与关联 2FA 同时更新时名称不同步、删除 2FA 后主账号仍残留密钥的问题。
- 修复关联 2FA 使用非默认算法、位数、周期或 HOTP 时，主账号详情生成错误验证码的问题。
- 修复明文旧备份导入后敏感账号与服务字段未重新加密的问题。
- 修复导入数据库失败后连接不可用，并防止同秒自动备份相互覆盖。
- 修复使用大写 `.DB` 扩展名时 SQLite 备份被误按 JSON 处理的问题。
- 修复账号搜索或平台筛选影响服务关联下拉框、2FA 分组及关联账号跳转的问题。
- 增加单实例锁，避免同一用户数据目录被多个应用进程并发打开。
- 限制账号和服务永久删除只能作用于回收站记录。
- 密码生成器改用系统加密安全随机数。

### 工程

- 统一界面偏好到主进程存储，并迁移旧的 localStorage 设置。
- 清理历史构建日志，完善忽略规则、测试与发布校验。
- 升级 Electron、Vite、electron-builder、better-sqlite3 等运行与打包依赖，移除已知高危依赖链。
- CI 使用 Node.js 22 LTS（最低 22.12），审计完整依赖树，并增加 Pull Request 核对模板。
- Release Action 在上传前校验安装包、blockmap 与 `latest.yml` 的版本、路径、大小和 SHA-512。
- GitHub Actions 升级到 Node.js 24 Runtime 对应主版本，移除 Node.js 20 弃用警告。
- README 增加基于纯演示数据生成的账号、服务、2FA 与统一回收站界面预览。
