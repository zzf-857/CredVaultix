# 安全模型

CredVaultix 是单机、离线优先的桌面管理工具，不上传遥测或凭据数据。

## 当前保护范围

- 账号用户名、密码、手机号、备用邮箱、账号 2FA 密钥、敏感自定义字段、TOTP/HOTP 密钥和敏感服务字段使用 AES-256-GCM 加密后写入 SQLite。
- Electron 开启渲染沙箱与 `contextIsolation`，关闭渲染进程的 Node.js 集成，并使用 CSP 与导航拦截限制外部内容。
- SQLite 导入前执行格式与完整性验证，替换前自动保存当前数据库副本。
- 任何服务表或 TOTP 静态加密迁移执行前都会先创建数据库副本。
- 服务和账号默认软删除，可从回收站恢复。

## 当前限制

- 当前版本尚未提供主密码或独立的保险库解锁流程。本地加密密钥由应用环境派生，主要用于避免数据库中的直接明文暴露，不能抵御已经取得当前 Windows 用户权限的攻击者。
- 普通备注、服务 URL 和非敏感字段按用户选择以明文保存。
- 复制到系统剪贴板的内容可能被同一系统会话中的其他程序读取；自动清理剪贴板仍属于候选功能。
- 正式安装包当前没有 Windows 代码签名证书。
- 当前备份不是带独立密码的跨设备加密归档；跨 Windows 用户迁移前应保留原环境并验证恢复结果。

主密码、自动锁定、可移植加密备份和剪贴板自动清理计划需在产品范围确认后实施。
